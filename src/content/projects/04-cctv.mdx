## Problem

HTTP 롱폴링 방식으로 서버와 통신하고 있어 LCP가 5.22초에 달했습니다. 매 갱신 주기마다 불필요한 네트워크 요청이 반복되었고, 실시간 CCTV 데이터를 지연 없이 확인하기 어려운 상황이었습니다.

## Approach

HTTP 폴링을 **WebSocket**으로 전환해 서버와 지속 연결을 유지하도록 개선했습니다. Chart.js로 실시간 센서 데이터를 시각화하고, 컴포넌트 단위 부분 업데이트를 구현해 불필요한 리렌더링을 최소화했습니다.

## Result

LCP **93% 개선 (5.22초 → 0.36초)**. 실시간 데이터 스트리밍 전환. 폴링 오버헤드 완전 제거. 사용자가 지연 없이 실시간 CCTV 영상 데이터를 확인할 수 있게 됐습니다.
